# BUISNESS_LOGIC

## Назначение продукта

Reflebot помогает проводить занятия, собирать рефлексию студентов после лекций и давать преподавателям и администраторам удобный способ просматривать результаты.

Продукт решает четыре основные задачи:

- зарегистрировать пользователя в боте и определить его роли;
- помочь администратору подготовить курс, лекции, вопросы и участников;
- дать преподавателю и администратору удобный доступ к аналитике;
- напомнить студенту о рефлексии и собрать её в формате кружков.

## Роли в системе

Один и тот же человек может одновременно быть:

- администратором;
- преподавателем;
- студентом.

После входа бот показывает человеку его роли и доступные действия.

Сейчас логика меню такая:

- администратор видит кнопки администратора;
- администратор также видит кнопки преподавателя, даже если отдельной роли преподавателя у него нет;
- преподаватель видит свои кнопки;
- студент без других ролей не видит стартовых кнопок.

## Общий принцип работы бота

Бот всегда ведёт пользователя пошагово.

На каждом шаге пользователь видит:

- текст с объяснением, что происходит сейчас;
- кнопки, если на этом шаге нужен выбор;
- ожидание текста или файла, если бот просит что-то ввести или загрузить.

Навигация строится вокруг кнопки `Назад`, списков с пагинацией и простых последовательных сценариев.

## Старт работы

### Сценарий входа

Когда пользователь пишет `/start`, система:

- находит его по Telegram username;
- определяет, кем он является в системе;
- при необходимости сохраняет или обновляет его `telegram_id`;
- показывает приветственное сообщение;
- перечисляет все найденные роли;
- показывает стартовое меню по ролям.

### Что видит пользователь после входа

Пользователь получает сообщение вида:

- успешная регистрация;
- его ФИО;
- список ролей;
- приглашение выбрать действие.

### Стартовые кнопки

Если есть роль администратора:

- `Создать администратора`
- `Создать курс`
- `Аналитика`
- `Ближайшая лекция`

Если есть только роль преподавателя:

- `Аналитика`
- `Ближайшая лекция`

Если есть только роль студента:

- кнопок нет.

## Сценарии администратора

### 1. Создание администратора

Кнопка:

- `Создать администратора`

Шаги:

- бот просит ввести ФИО нового администратора;
- после этого бот просит ввести Telegram username без `@`;
- если username некорректный, бот просит ввести его заново;
- если такой администратор уже есть, бот предлагает ввести другой username;
- после успешного создания бот возвращает пользователя в главное меню.

Что отображается после успеха:

- сообщение о том, что администратор успешно создан;
- снова показывается главное меню.

### 2. Создание курса

Кнопка:

- `Создать курс`

Шаги:

- бот просит загрузить Excel-файл с расписанием курса;
- после загрузки система создаёт курс и лекции;
- затем показывает карточку созданного курса.

Что отображается после успешного создания курса:

- название курса;
- дата начала курса;
- дата окончания курса;
- вопрос “Что дальше?”.

Кнопки после создания курса:

- `Спаршенные лекции`
- `Привязать преподавателей`
- `Отменить парсинг`

### 3. Отмена парсинга курса

Кнопка:

- `Отменить парсинг`

Результат:

- курс и его лекции удаляются;
- пользователь возвращается в главное меню;
- бот показывает сообщение, что парсинг курса отменён.

### 4. Просмотр спаршенных лекций

Кнопка:

- `Спаршенные лекции`

Что отображается:

- название курса;
- список лекций;
- пагинация по 5 лекций на страницу.

Кнопки на экране:

- отдельная кнопка на каждую лекцию;
- `Предыдущая страница`, если это не первая страница;
- `Следующая страница`, если это не последняя страница;
- `Назад`.

### 5. Карточка лекции

После выбора конкретной лекции бот показывает:

- тему лекции;
- дату лекции;
- время начала и окончания;
- количество вопросов;
- загружена ли презентация;
- загружена ли запись лекции.

Кнопки в карточке лекции:

- `Изменить тему`
- `Изменить дату`
- `Вопросы`
- `Презентация`
- `Запись лекции`
- `Назад`

### 6. Изменение темы лекции

Кнопка:

- `Изменить тему`

Шаги:

- бот просит ввести новую тему;
- после ввода обновляет тему;
- возвращает пользователя обратно в карточку лекции.

### 7. Изменение даты и времени лекции

Кнопка:

- `Изменить дату`

Шаги:

- бот просит ввести новую дату и время в формате `ДД.ММ.ГГГГ ЧЧ:ММ-ЧЧ:ММ`;
- если формат неверный, просит повторить ввод;
- если время окончания раньше времени начала, просит повторить ввод;
- после успешного ввода обновляет дату и время;
- возвращает пользователя обратно в карточку лекции.

### 8. Управление вопросами к лекции

Кнопка:

- `Вопросы`

Что отображается:

- список вопросов к лекции с порядковыми номерами;
- если вопросов нет, бот так и пишет.

Кнопки:

- `Добавить`
- `Изменить`
- `Удалить`
- `Назад`

#### Добавление вопроса

Кнопка:

- `Добавить`

Шаги:

- бот просит ввести текст вопроса;
- после ввода добавляет вопрос;
- возвращает в экран списка вопросов.

#### Изменение вопроса

Кнопка:

- `Изменить`

Шаги:

- бот просит ввести номер вопроса и новый текст в одном сообщении;
- после ввода обновляет вопрос;
- возвращает в экран списка вопросов.

#### Удаление вопроса

Кнопка:

- `Удалить`

Что отображается:

- кнопки со всеми вопросами;
- при нажатии на нужный вопрос он удаляется;
- затем бот возвращает в экран списка вопросов.

### 9. Управление презентацией лекции

Кнопка:

- `Презентация`

Что отображается:

- если презентация уже есть, бот показывает, что она загружена;
- если презентации нет, бот показывает, что она не загружена.

Кнопки:

- `Скачать презентацию`, если файл уже загружен;
- `Изменить презентацию`;
- `Назад`.

#### Загрузка или замена презентации

Кнопка:

- `Изменить презентацию`

Шаги:

- бот просит прислать файл;
- после загрузки сохраняет его;
- возвращает пользователя в экран презентации.

#### Скачивание презентации

Кнопка:

- `Скачать презентацию`

Результат:

- бот отправляет файл пользователю в Telegram;
- после этого остаётся экран презентации.

### 10. Управление записью лекции

Кнопка:

- `Запись лекции`

Что отображается:

- если запись есть, бот показывает, что она загружена;
- если записи нет, бот показывает, что она не загружена.

Кнопки:

- `Скачать запись`, если файл уже загружен;
- `Изменить запись`;
- `Назад`.

#### Загрузка или замена записи

Кнопка:

- `Изменить запись`

Шаги:

- бот просит прислать файл;
- после загрузки сохраняет его;
- возвращает пользователя в экран записи.

#### Скачивание записи

Кнопка:

- `Скачать запись`

Результат:

- бот отправляет запись пользователю;
- после этого остаётся экран записи.

### 11. Привязка преподавателей к курсу

Кнопка:

- `Привязать преподавателей`

Шаги:

- бот просит ввести ФИО преподавателя;
- затем просит ввести его Telegram username без `@`;
- после этого преподаватель привязывается к курсу.

Что отображается после успеха:

- сообщение о том, что преподаватель успешно добавлен к курсу.

Кнопки:

- `Добавить ещё преподавателя`
- `Привязать студентов`

#### Добавить ещё преподавателя

Кнопка:

- `Добавить ещё преподавателя`

Результат:

- запускает тот же сценарий ввода ФИО и username для следующего преподавателя.

### 12. Привязка студентов к курсу

Кнопка:

- `Привязать студентов`

Шаги:

- бот просит загрузить CSV-файл со студентами;
- после загрузки система добавляет студентов и привязывает их к курсу и лекциям.

Что отображается после успеха:

- количество добавленных студентов;
- затем пользователь возвращается в главное меню.

## Сценарии преподавателя и администратора

Администратор и преподаватель работают с одинаковыми разделами аналитики.

Разница такая:

- преподаватель видит только свои курсы и свои ближайшие лекции;
- администратор видит все курсы и ближайшую лекцию по всей системе.

### 1. Ближайшая лекция

Кнопка:

- `Ближайшая лекция`

Что отображается:

- тема ближайшей будущей лекции;
- дата;
- время.

Кнопки:

- `Назад`

Если будущих лекций нет:

- бот пишет, что запланированных лекций нет.

### 2. Аналитика: выбор курса

Кнопка:

- `Аналитика`

Что отображается:

- экран выбора курса;
- список курсов с пагинацией по 5 на страницу.

Кнопки:

- отдельная кнопка на каждый курс;
- `Предыдущая страница`, если нужно;
- `Следующая страница`, если нужно;
- `Назад`.

### 3. Аналитика: меню курса

После выбора курса бот показывает:

- название курса;
- предложение выбрать действие.

Кнопки:

- `Статистика по лекции`
- `Найти студента`
- `Назад`

### 4. Аналитика: статистика по лекции

Кнопка:

- `Статистика по лекции`

Что отображается:

- список лекций выбранного курса;
- пагинация по 5 лекций на страницу.

Кнопки:

- отдельная кнопка на каждую лекцию;
- кнопки пагинации;
- `Назад`.

#### Карточка статистики по лекции

После выбора лекции бот показывает:

- тему лекции;
- дату и время лекции;
- сколько студентов прикреплено к лекции;
- сколько студентов отправили рефлексию;
- сколько ответов на вопросы отправлено;
- список студентов, которые отправили рефлексию.

Кнопки:

- отдельная кнопка на каждого студента с рефлексией;
- кнопки пагинации;
- `Назад`.

#### Просмотр рефлексии конкретного студента из статистики лекции

После выбора студента бот показывает уже не краткую статистику, а готовый диалог рефлексии:

- имя студента;
- тему лекции;
- дату отправки;
- затем все кружки общей рефлексии по порядку;
- затем каждый вопрос;
- затем кружки ответа на этот вопрос.

Это выглядит как последовательность сообщений, чтобы можно было просмотреть полный диалог студента по конкретной лекции.

Кнопки:

- `Назад`

### 5. Аналитика: поиск студента

Кнопка:

- `Найти студента`

Что отображается:

- список студентов выбранного курса;
- пагинация по 5 студентов на страницу.

Кнопки:

- отдельная кнопка на каждого студента;
- кнопки пагинации;
- `Назад`.

#### Карточка статистики по студенту

После выбора студента бот показывает:

- имя студента;
- сколько лекций у него по данному курсу;
- сколько рефлексий он отправил по этому курсу;
- сколько ответов на вопросы он отправил по этому курсу;
- список лекций, по которым у него есть рефлексия.

Кнопки:

- отдельная кнопка на каждую лекцию с рефлексией;
- `Назад`.

#### Просмотр рефлексии студента из карточки студента

После выбора лекции бот показывает тот же полный диалог:

- общая рефлексия по лекции;
- вопросы;
- ответы на вопросы;
- всё в правильном порядке.

Кнопки:

- `Назад`

## Сценарии студента

### 1. Вход в бота

Если у пользователя только роль студента, после `/start` он видит:

- приветствие;
- своё ФИО;
- роль студента;
- без стартовых кнопок.

Основной сценарий студента запускается не из главного меню, а из напоминания после лекции.

### 2. Напоминание о рефлексии после лекции

После завершения лекции студенту может прийти отдельное сообщение:

- приветствие;
- просьба отправить рефлексию по конкретной лекции;
- название лекции.

Кнопка:

- `Записать кружок`

### 3. Начало рефлексии по лекции

После нажатия `Записать кружок` бот пишет:

- `Записывайте кружок, я вас слушаю.`

На этом шаге кнопок нет.

Бот ждёт от студента кружок.

### 4. После записи кружка по лекции

Когда студент отправляет кружок, бот пишет:

- `Кружок записан, что хотите сделать?`

Кнопки:

- `Отправить рефлексию`
- `Удалить кружок`
- `Добавить ещё один кружок`

### 5. Добавление ещё одного кружка

Кнопка:

- `Добавить ещё один кружок`

Результат:

- бот снова пишет `Записывайте кружок, я вас слушаю.`;
- ждёт следующий кружок;
- затем снова показывает экран с тремя кнопками.

### 6. Удаление кружка

Кнопка:

- `Удалить кружок`

Логика такая:

- удалить можно только последний ещё не отправленный кружок;
- если после удаления кружки ещё остались, бот пишет `Кружок удалён.` и оставляет экран с тремя кнопками;
- если удалён последний кружок и больше черновиков нет, бот возвращает самый первый экран рефлексии с кнопкой `Записать кружок`.

### 7. Отправка рефлексии по лекции

Кнопка:

- `Отправить рефлексию`

Результат зависит от наличия вопросов у лекции.

Если вопросов нет:

- бот пишет `Спасибо за рефлексию.`
- сценарий завершается.

Если вопросы есть:

- бот начинает задавать их по одному;
- после этого сценарий продолжается уже как сбор ответов на вопросы.

### 8. Ответ на вопрос по лекции

Когда начинается блок вопросов, бот показывает:

- номер вопроса;
- общее число вопросов;
- текст вопроса;
- просьбу записать кружок с ответом.

Кнопка:

- `Записать кружок`

После нажатия бот снова переходит в режим ожидания кружка.

### 9. После записи кружка-ответа на вопрос

Когда кружок по вопросу записан, бот снова показывает:

- `Кружок записан, что хотите сделать?`

Кнопки:

- `Отправить рефлексию`
- `Удалить кружок`
- `Добавить ещё один кружок`

Смысл кнопок тот же:

- можно добавить ещё один кружок к ответу на текущий вопрос;
- можно удалить только последний кружок из черновика текущего ответа;
- можно отправить ответ и перейти к следующему вопросу.

### 10. Завершение сценария вопросов

Когда вопросы заканчиваются, бот пишет:

- `Вопросов больше нет. Спасибо за рефлексию.`

После этого сценарий завершён.

## Что получает преподаватель и администратор в аналитике

С точки зрения продукта аналитика отвечает на два вопроса:

- что происходило на конкретной лекции;
- что делал конкретный студент в рамках курса.

В обоих случаях итогом можно открыть полный “диалог” студента:

- сначала общая рефлексия по лекции;
- потом ответы на вопросы;
- каждый вопрос предваряется своим текстом;
- все кружки показываются в исходном порядке.

## Файлы в продуктовой логике

Для пользователя важны три вида файлов:

- презентация лекции;
- запись лекции;
- кружки студента.

Пользовательский смысл такой:

- администратор может загружать и обновлять презентацию и запись;
- администратор может запросить, чтобы бот отправил эти файлы обратно в чат;
- студент отправляет кружки как часть рефлексии;
- преподаватель и администратор могут просматривать кружки в аналитике.

## Навигация

Во всех сценариях пользователь двигается по цепочке экранов.

Общие правила:

- если экран является списком, там обычно есть пагинация;
- кнопка `Назад` возвращает на предыдущий экран;
- после завершения большого сценария пользователь либо возвращается в карточку сущности, либо попадает обратно в главное меню.

## Ошибки и ограничения, важные с точки зрения продукта

- если пользователь вводит неверный username, бот просит повторить ввод;
- если пользователь вводит неверный формат даты, бот просит повторить ввод;
- если у пользователя нет прав на действие, бот сообщает об этом;
- если возникает техническая ошибка, бот показывает безопасное сообщение без внутренних деталей;
- студент не может повторно отправить рефлексию по той же лекции после завершения сценария;
- аналитика доступна преподавателю только по его курсам, а администратору по всем курсам.

## Краткая пользовательская картина

Система работает так:

- администратор готовит курс, лекции, вопросы и участников;
- преподаватель ведёт занятия и затем смотрит аналитику;
- после лекции студент получает напоминание и отправляет рефлексию кружками;
- ответы собираются сначала по всей лекции, затем по вопросам;
- преподаватель и администратор могут открыть результаты и просмотреть их как живой диалог.

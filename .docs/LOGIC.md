При /start происходит поиск в бд, является ли пользователь админом, учителем, студентом по username. Пользователь может быть всем одновременно, если telegram_id null, то его обновить, сделать таким, какой пришёл в /start. 

Везде сообщение в ответ одно и то же - Привет, ФИО, регистрация прошла успешно, ваши роли: тут вывести роли того, кем он является

Далее нужно вывести кнопки:

Если пользователь админ:Создать админа, создать курс

Если пользователь учитель:

Аналитика, Ближайшая лекция

Если студент:

Кнопок нет

Кнопка создать курс: пишем Отправьте xlsx файл, для парсинга расписания. xlsx файл должен иметь следующие колонки: Название, Тема лекции, Дата, Время.

Кнопок тут нет, просто ждём отправки файла. 

После отправки файла: парсим файл, добавляем курс и лекции в бд. Парсинг уже реализован, единственное в чём может быть проблема- возможно как-то работают с полем колонки преподаватель, которое раньше было. Если было - убрать. Выводим после парсинга: парсинг успешно завершён для курса: Название, дата начала, дата конца. Внизу кнопки: спаршенные лекции, привязать преподавателей, отменить парсинг.

При нажатии на отменить парсинг, данный курс и лекции удаляются, возвращаемся назад.

При нажатии на спаршенные лекции выводить в кнопках каждую лекцию, относящуюся к данному курсу с пагинацией по 5, action - info\_for\_create\_lection и кнопка назад. При нажатии на кнопку выводится информация лекции в сообщении: Тема лекции, время когда произойдёт, вопросы к данной лекции, есть ли презентация, есть ли запись лекции. Далее кнопки: Изменить тему, изменить дату, вопросы, презентация, запись лекций, назад.

При изменении темы - просто просим ввести новую, обновляем после ввода, возвращаем обратно где информация лекции, так же должна быть кнопка назад. Аналогичная логика с изменить дату. При нажатии на кнопку вопросы - выводятся все запросы к данной лекции и ниже 3 кнопки, добавить, удалить, изменить. При добавлении просто ввод одного вопроса текстом, при изменении в начале цифра в виде порядкового номера, как вывелись вопросы, при удалении вывести вопросы в виде кнопок и при нажатии удалить, после выполнения любого из действий этих возвращать в вопросы и отображать новые. При нажатии презентация выводим имя презентации и ниже кнопки: изменить презентацию, скачать презентацию, при нажатии на скачать - выдаём файл, возвращаем инфу о презентации, при нажатии на изменить презентацию, ждём получения нового файла, его скачиваем. Аналогично с записью лекций. Кнопка назад возвращает на место, где были все лекции

При нажатии на привязать преподавателей, пишем: введите ФИО и username учителя. Ждём когда введёт, после ввода парсим добавлем в Teachers, TeachersCourses, TeachersLections, выводим: ФИО успешно добавлено, ниже кнопки: Добавить ещё преподавателя, привязать студентов. Кнопка добавить ещё преподавателя работает аналогично. Кнопка привязать студентов, выводим: отправьте csv файл со студентами с полями: ФИО, username. После отправки файла, парсим файл. Добавляем Students, StudentsCourses, StudentsLections. Выводим было добавлено: n студентов. Кнопки такие же, какие выводятся после /start

При нажатии на кнопку Ближайшая лекция надо сделат action - info\_for\_create\_lection

После завершения лекции студенту может прийти сообщение:

- Здравствуйте, отправьте рефлексию по лекции: название лекции.

И кнопка:

- Записать кружок
- action: `student_start_reflection:<lection_session_id>`

Дальше workflow такой:

1. После нажатия кнопки backend отвечает: `Записывайте кружок, я вас слушаю.`
2. Бот ждёт кружок и отправляет `telegram_file_id` в `POST /actions/file`
3. Backend отвечает: `Кружок записан, что хотите сделать?`
4. Кнопки:
   - Отправить рефлексию
   - Удалить кружок
   - Добавить ещё один кружок
5. Удалять можно только последний ещё не отправленный кружок
6. Если все кружки удалены, backend возвращает исходный prompt по лекции
7. При отправке рефлексии backend сохраняет данные в `LectionReflection` и `ReflectionVideo`
8. Если у лекции есть вопросы, backend задаёт их по одному и повторяет ту же логику с кружками
9. После последнего вопроса backend сохраняет данные в `LectionQA` и `QAVideo`
10. Финальное сообщение:
   - `Вопросов больше нет. Спасибо за рефлексию.`

При нажатии на кнопку Аналитика, выводится: Выберите сессию курса, внизу по кнопкам с пагинацией в 5 кнопок выводятся курсы. Курсы для админа доступны все, для учителя только те, к которым он привязан в TeachersCourses. После нажатия на нужный курс выводится Название курса и ниже 3 кнопки. Статистика по лекции, Найти студента и назад. При нажатии на кнопку Статистика по лекции, выводятся в кнопках с пагинацией все лекции, привязанные к этому курсу и назад. При нажатии на конкретную лекцию выводится другое, не то, что было раньше. Выводится Тема лекции, дата лекции, вопросы к данной лекции, есть ли презентация, есть ли запись лекции, сколько студентов прикреплены к данной лекции, сколько студентов отправили рефлексию к данной лекции, сколько студентов отправили ответы на вопросы к данной лекции. В виде кнопок ФИО студентов, которые отправили рефлексию к данной лекции с пагинацией и кнопка назад. При нажатии на данную кнопку выводится как диалог между студентом и рефлексией и вопросами и рефлексией. Логику этого реализуем позже.

При нажатии на кнопку Найти студента, в кнопках выводятся все студенты, привязанные к данному курсу. Далее при нажатии на студента, выводится статистика: записан на n лекций по данному курсу, отправил n рефлексий по данному курсу (считать именно LectionReflection, а не ReflectionVideo), отправил n ответов на вопросы по данному курсу (считать именно LectionQA). В виде кнопок: лекции, на которые студент дал рефлексию. При нажатии на данную кнопку выводится как диалог между студентом и рефлексией и вопросами и рефлексией. Логику этого реализуем позже.
